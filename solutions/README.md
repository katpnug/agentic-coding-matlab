# Solutions

Reference implementations for modules 1–4, plus the figures they produce
(`expected_figures/`). Your agent-generated code will not (and need not) match
these line for line — compare *results*: same printed numbers where the
module specifies them, figures that show the same structure.

| Module | Script(s) |
|---|---|
| 1 — warm-ups | `module1/warmup1_fixed.m`, `warmup2_readable.m`, `warmup3_nice_figure.m`, `warmup4_fast.m` |
| 2 — toy analysis | `module2/speed_tuning.m` |
| 3 — reach kinematics | `module3/reach_kinematics.m` |
| 4 — event-aligned spikes | `module4/reach_psth.m` |

Each script is self-contained: open it in MATLAB and run it (it `cd`s to the
repo root itself), or ask your agent to run it with the MATLAB tools. Module 0
and 5 have no solution scripts — their outcomes are a working setup and
context files you write yourself.

Instructors: these scripts double as a smoke test of the repo. If all four
run cleanly on a fresh clone, the data subset and environment are good.
